Project Introduction

This project aims to improve the reCAPTCHA from CAPTCHA system that helps web hosts to distinguish between human and automated access to websites. Usually, reCAPTCHA will ask users to identify hard-to-read random text/numbers or match images with the given objects. This link leads to a reCAPTCHA demo

Triggers

Users tap the image to select the images.

Rules

  • Users need to determine ALL images that contains the object in order to go to the next step.
  • Refreshing the page will change the topic for choosing images.
  • Users will need to reselect the images if they select the ONE image wrong.
  • The topic will automatically change to a different objects when users select wrong more than 3 times.

    • Feedbacks

    When users tap the image:

  • The selected image gets smaller
  • The white border appears
  • The ticking arrow appears to confirm the users that the image is selected.

    • LOOPS & MODES

    When users select the wrong images more than 3 times, a new section with new topic and images will appear for users to select until the users get it right.

    My improvements

    TRIGGERS (INPUT BOX)

    Firstly, when users access a website that requires reCatpcha, a card will appear asking the users to fill out a necessary text to the input box. After filling out, the users will need to click the "I am not a robot" button to verify that they are not robots.

    FEEDBACK (IMAGES CARD)

    Secondly, after users fill out the input box, a card will appear with the instructions guiding users to select all images that have a specific topic.

    SELECT ALL IMAGES

    Thirdly, after viewing all images, users will tap to select the image they believe is related to the topic. After users tap the images, the selected image will turn darker with an "X" icon appear to notify users that there image is selected.
